Left Recursion

Every left-recursive sequence after inlining makes the following transformation:

a <- a rest1 / a rest2 / s1 / s2 / a rest3
a <= (s1 / s2) a2
a2<= (rest1 / rest2 / rest3) a / succ

s1 and s2 are assigned all tag that are already assigned to rest1, rest2, and rest3 - so that they collapse properly when one of the rest* expressions is invoked.

Definitions

Expressions
call

some_rule - A reference to another rule to be invoked.

seq

a b c - Sequence. A space delimited list of sub-expressions, invoked sequentially, that fails if any of the sub-expressions fail.

ord

a/b/c - Ordered choice. A forward-slash delimited sequence of sub-expressions that are evaluated only until one succeeds. A prioritized choice sequence.

xord

a//b//c - Exclusive ordered choice. A specialized ord where the first sub-expression to begin to succeed determines the overall result of the expression. Intended to be used only internally by the system.

char

[a-z_] - Character class, matches one/any of the ranges/singles.

lit

'abc' - Sequence of characters to consume.

Special Expressions
succ

succ - (Special) Always succeeds consuming nothing- also known as the empty string.

fail

fail - (Special) Always fails, consuming nothing.

any

. or any - (Special) Matches and consumes the next available input.

eof

!. or eof - (Special) Matches the end of all input.

Misc Expressions
e

Shorthand for expression.

term

Shorthand for sub-expressions (equivalent to e in reality).

nop

Shorthand for an expression or sub-expression that can be safely removed.

body

The "main" or toplevel term for a rule.

t()

Transformation function that user supplies to process partial results

c()

Collapse function - internally generated expression that takes everything off the current accumulator and assigns it into specified tag buckets.

Attributes
Prefixes / Predicates
notp

!e1 - Not predicate, fails if e1 succeeds, consumes nothing

andp

&e1 - And predicate, succeeds if e1 succeeds, consumes nothing

Suffixes / Quantifiers
star

e1* - Kleene star, greedily consumes zero or more of e1, always succeeds

plus

e1+ - Greedily consumes 1 or more of e1, failing if there are none.

opt

e1? - Consumes zero or 1 of e1, always succeeds.

PEG First Normal Form

Applicable to all (all)

  1. Every call inlined except entry-points, recursive calls, and accumulators.

  2. Attributes on all expressions are normalized / reduced (see below)

  3. All + expressions normalized to * expressions. (only after attributes fully normalized).

  4. No potential degenerate loops - or warnings emitted if applicable.

Ordered Choice (ord)

  1. Contains no ord terms. (TODO: invalidated by certain term attributes).

  2. Contains no succ terms. (Everything after removed and ord rewritten to ord?).

  3. Contains nothing after a star term (which always succeeds).

  4. Contains no opt terms. (Rewritten as term/succ and reduced as per #2).

  5. Contains no fail terms. (Safely removed as nops).

  6. Contains at least two terms. (Single term replaces nop ord, with attributes combined).

  7. Contains no duplicate or equivalent terms. (Leftmost is kept).

  8. Contains no term2 to the right of term1 that have term1 as a prefix.

  9. All inner terms are in reduced form.

Sequences (seq)

  1. Contains no seq term except where attributes differentiate. (TODO).

  2. Contains no succ terms. (Removed because they are nops).

  3. Contains no fail terms. (A seq with a fail term is itself a fail term).

  4. Contains at least two terms. (Single term replaces nop seq, with attributes combined).

  5. Contains no two identical star terms in a row (safely remove second)

  6. All inner terms are in reduced form.

Attributes (attr)

One should recursively apply the rule combinations from innermost to outermost, stopping when all are processed or when an error is encountered.

  1. opt Group

    1. (e1?)? ⇒ e1?

    2. (e1?)* ⇒ Error: Degenerate loop at first char where e1 fails (intended?: e1*)

    3. (e1*)? ⇒ e1*

    4. (e1?)+ ⇒ Error: Degenerate loop at first char where e1 fails (intended?: e1+)

    5. (e1+)? ⇒ e1*

    6. &(e1?) ⇒ succ

    7. (&e1)? ⇒ succ

    8. !(e1?) ⇒ fail

    9. (!e1)? ⇒ succ

    10. any? ⇒ Valid, no change

    11. eof? ⇒ succ

    12. succ? ⇒ succ

    13. fail? ⇒ succ

  2. star Group

    1. (e1*)* ⇒ Error: Degenerate loop

    2. (e1*)+ ⇒ fail (e1* never leaves anything for + to consume)

    3. (e1+)* ⇒ e1*

    4. &(e1*) ⇒ succ

    5. (&e1)* ⇒ Error: Degenerate loop

    6. !(e1*) ⇒ fail

    7. (!e1)* ⇒ Error: Degenerate loop when e1 fails

    8. any* ⇒ Valid, no change

    9. eof* ⇒ Error: Degenerate loop ((!any)*)

    10. succ* ⇒ Error: Degenerate loop

    11. fail* ⇒ fail

  3. plus Group

    1. (e1+)+ ⇒ fail (never anything for the second to consume)

    2. &(e1+) ⇒ Valid, no change

    3. (&e1)+ ⇒ Error: Degenerate loop when e1 succeeds

    4. !(e1+) ⇒ Valid, no change

    5. (!e1)+ ⇒ Error: Degenerate loop when e1 fails

    6. any+ ⇒ Valid, no change

    7. eof+ ⇒ Error: Degenerate loop

    8. succ+ ⇒ Error: Degenerate loop

    9. fail+ ⇒ fail

  4. andp Group

    1. &(&e1) ⇒ &e1

    2. !(&e1) ⇒ !e1

    3. &(!e1) ⇒ !e1

    4. &any ⇒ Valid, no change

    5. &eof ⇒ Valid, no change

    6. &succ ⇒ succ

    7. &fail ⇒ fail

  5. notp Group

    1. !(!e1) ⇒ &e1

    2. !any ⇒ eof

    3. !eof ⇒ any

    4. !succ ⇒ fail

    5. !fail ⇒ succ

Detecting Degenerate Loops

Expression Classification

Definitions
0

Success, nothing consumed

1

Success, something consumed

f

Failure, nothing consumed

s

0 or 1 (success, sometimes something consumed)

a

s or f (any of the above)

  1. succ → 0

  2. any/char/lit → 1 | f

  3. call → (whatever its expression reduces to)

  4. e1 e2 → 0 if e1→0 and e2→0

  5. e1 e2 → 1 if e1→1 and e2→s (or visa versa)

  6. e1 e2 → f if e1→f or (e1→s and e2→f)

  7. e1/e2 → s if e1→s

  8. e1/e2 → a if e1→f and e2→a

  9. e* → 1 if e →1

  10. e* → 0 if e →f

  11. !e → f if e →s

  12. !e → 0 if e →f

  13. e1/e2* → s

  14. e1/e2? → s

  15. e1/succ → s

Algorithm

The primary objective is to detect any concrete possibility of e1→0 in e1*. The secondary objective, if possible, is to detect the possibility of a recursive rule recursing without consuming any input (after left-recursion rules are put into place).

PEG Second Normal Form

(In preparation for all transformations)

  1. No lit expressions - only sequences of (small) char ranges.

  2. Every char range with multiple ranges is instead an ord of each range/char.

  3. star expressions pulled into their own tail-recursive rules.

  4. Only recursive rules (including what used to be star expressions) are not inlined.

PEG Optimized Form

(All transformations have finished)

  1. (If possible) no ord expressions- only xord expressions in their place.

  2. All potentially left-recursive rules are marked for compiler generator.

  3. Sequences of char recombined into lit form where possible. (NOTE: the transformation function may need to break it back up- but it’ll still make matching faster…)

  4. Predicates reference new (optimized) paths - predicates do not reference any rules referenced by any non-predicate rules.

  5. Predicates do not attempt to consume more of the input once success or failure is certain.

XORD creation

  1. R <- ([x]//[y]) R / succ can be rewritten as R <- ([x]//[y]) R // succ - TODO: figure out how it is discoverable (in this case first term succeeds or fails, and second one always succeeds, so only first one is determinant…)
